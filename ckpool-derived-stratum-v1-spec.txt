ckpool Stratum V1 share protocol

1. Subscribe
   Client sends mining.subscribe.
   Server returns:
     result = [subscriptions, extranonce1, extranonce2_size]

   extranonce1:
     hex string, opaque byte sequence
   extranonce2_size:
     integer byte count

2. Authorize
   Client sends mining.authorize with workername and password.

3. Notify
   Server sends mining.notify with params:
     [job_id, prevhash, coinbase1, coinbase2, merkle_branch[], version, nbits, ntime, clean_jobs]

4. Coinbase reconstruction
   coinbase = hex(coinbase1) || hex(extranonce1) || hex(extranonce2) || hex(coinbase2)

5. Merkle root reconstruction
   root = dsha256(coinbase)
   for each branch in merkle_branch:
       root = dsha256(root || branch)
   root is then word-flipped before insertion into the header buffer.

6. Header reconstruction
   Header fields:
     version(4) || prevhash(32) || merkle_root(32) || ntime(4) || nbits(4) || nonce(4)

   ckpool patches:
     merkle_root at offset 36
     ntime       at offset 68
     nonce       at offset 76

   nonce submit string is hex-decoded directly into the 4 header bytes.
   ntime submit string is parsed as uint32 and written as big-endian bytes.

7. Submit
   Client sends:
     mining.submit(workername, job_id, extranonce2, ntime, nonce [, version_mask])

   Constraints:
     extranonce2 length must equal extranonce2_size bytes
     nonce must be 8 hex chars
     ntime must satisfy original_ntime <= ntime <= original_ntime + 7000

8. Acceptance
   Pool reconstructs the candidate block header from the submitted values and computes share difficulty.
   Share is accepted only if the computed hash meets the active share target.
